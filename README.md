# DESCRIPTION
The design of a 4-stage pipelined processor represents a fundamental leap in computer architecture, transitioning from a single-cycle system to one that leverages parallelism by overlapping instruction execution stages. In this specific implementation, the instruction cycle is decomposed into four distinct logical segments: Instruction Fetch (IF), Instruction Decode (ID), Execute (EX), and Write Back (WB). By partitioning the hardware into these specific zones, the processor is capable of working on four different instructions simultaneously, effectively increasing the throughput—the number of instructions completed per clock cycle—without necessarily increasing the clock frequency. The primary goal is to ensure that while one instruction is being committed to memory, the next is being processed by the Arithmetic Logic Unit (ALU), and subsequent ones are being decoded or fetched from the instruction memory.
The cycle begins at the Instruction Fetch (IF) stage, where the Program Counter (PC) acts as a pointer to the instruction memory. On every positive clock edge, the processor retrieves a 16-bit machine code and advances the PC. This raw instruction is then passed into the Instruction Decode (ID) stage. Here, the control logic "slices" the 16-bit word to identify the operation to be performed—such as ADD, SUB, or LOAD—and identifies the target registers. The processor accesses the internal register file during this stage to pull the necessary operands, preparing them for calculation. Because this is a pipelined design, the data is stored in intermediate "pipeline registers" (IF/ID and ID/EX). These registers act as physical barriers that hold the data steady so that the next stage can process it without being affected by the new instruction entering the pipeline behind it.
The third stage, Execute (EX), is where the actual computation occurs. For arithmetic instructions like ADD and SUB, the ALU performs the binary operation on the values retrieved during the decode phase. For the LOAD instruction, this stage interfaces with the data memory to fetch a value from a specific address. Finally, the Write Back (WB) stage completes the lifecycle of the instruction. It takes the result generated by the ALU or the data retrieved from memory and writes it back into the destination register in the register file. This is a critical moment of architectural state change.
A key challenge in this design is the management of data hazards. In a 4-stage pipeline, an instruction may require the result of a previous instruction that has not yet reached the Write Back stage. This implementation handles such "Read-After-Write" (RAW) hazards through the use of software-level No-Operation (NOP) insertions, ensuring the pipeline is cleared before a dependent calculation begins. By simulating this design in a tool like EDA Playground using Verilog, one can visualize the flow of data through the waveform viewer. This functional design serves as a robust proof of concept for RISC-style processing, demonstrating how modular hardware stages can work in harmony to execute complex sequences of instructions with high efficiency and predictable timing.
